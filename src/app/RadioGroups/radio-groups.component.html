Radio buttons are often used in groups where only one option can be selected at a time. The expected navigation is for a user to tab into the first radio element, where the name <label> will be read, followed by the role type. It will also read the index of that raido and its associated <legend> or group label. Users can then use arrow keys to traverse the group. Each radio button requires a label for the button, and some sort of group label to help the user understand the context.

How to make a radio group accessible?

  • Starting from the top, a radio group must have a visible <legend>. These are curicual for the understanding and purpose of the radio group. They can be seen as the instructions or the group label.
  • Similar to many other elements, each individual radio button must have a visible label. All users need to know what each button represents.
  • The name, role, state, & value must be programmatically determinable. (role is often baked into the native HTML <button> component | name is the visible text | value, state are required for more complex functionality)

This first radio group is using the bare mininmum to be accessible. There is a group label, and each radio button has a visible label. Only 2 props are required to pass accessibility when using cel-components.
