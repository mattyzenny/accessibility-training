Input Validation

Input field validation is important for accessibility. Without the correct semantic association, a screen reader may not know which input the error is referring to or even worse, they may not be aware of an error at all.

            <cel-text-field data-field-label="Last Name" data-required="true" id="cel-textField-7"
          
          
            (blur)=
          "validateInputOnBlur('cel-textField-7')"
          
          
            [errorMessage]=
          "errorMessages['cel-textField-7']"> </cel-text-field>

            (blur)
            This is an onBlur() function. It runs error validation when a user tabs out or clicks out of the element.
          
      • 
          
    • 
            [errorMessage]
            This is an array of error messages that will be displayed if the input is invalid.
            This should be added to any required input that need validation.

  2. 
        Here we add the [successMessage] which behaves exactly like the [errorMessage]. It
        takes an array of messages.

            data-toggle-password
            This attribute is used to toggle the password visibility. If you inspect the code, the
            data-toggle-password is bound to the input type.
            On toggle, it is setting the input type to text or password.

            data-autocomplete
            Mentioned in previous examples, the autocomplete helps with auto-filling stored data.
