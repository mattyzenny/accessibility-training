Buttons are an important part of Design. They are used in just about every page within Realize. By inheriting a cel-button, you will want to make sure that all buttons are accessible.

What makes an accessible button?

  • A button must have a visible label.
  • If a button has an accessible label, it must start with the visible text.
  • The name, role, state, & value must be programmatically determinable. (role is often baked into the native HTML <button> component | name is the visible text | value, state are required for more complex functionality)

This button is using only the label attribute. Though this button has very little functionality, it is a fully accessible component. Turn on the AT and navigate to the element.

Button1 

          <cel-button id="button0" data-label="Cancel"> </cel-button>
        

      

    

  

  

    

      When you navigate to the button, the role 'button' and the name
      'cancel' are announced. There are no states or values assigned to this button, so they are not
      needed.
    

  






  

    

      This button is using HAT, hidden accessible text. This means it is providing a different
      label to screen reader users.
      

      To be accessible, we must provide the same information to all users. Therefore it is important use hidden text
      sparingly and only when providing supplemental information.
    

  

  

    

      
    

  

  

    

      

        
      

      

        

          Button2
          

          <cel-button id="button1" 
            data-label="Cancel"  data-button-aria-label=
          "Cancel, we are using data-aria-label to pass hidden accessible text to the user"></cel-button>
        

      

    

  

  

    

      The name  'data-button-aria-label' must first start with the visible label, and then
      provide supplemental information.
    

  






  

    

      When this button is clicked we have a UI change. If you click again, the UI changes back. This button has toggle functionality and must pass that visual state change to screen reader users.
    

  

  

    

      
    

  

  

    

      

        
      

      

        

          Button3
          

          <cel-button id="button2" 
            data-label="Cancel"  data-toggle=
          "true"></cel-button>
        

      

    

  

  

    

      This button has name  'cancel', role  'button',
      state  'selected'.
