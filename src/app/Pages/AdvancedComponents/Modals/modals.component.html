Intro

Modals should be accessible to all users, including those using screen readers and keyboards. They should provide clear focus management, accessible labeling, and proper interaction handling.

Required

  • Modals should start with a <h2>.
  • Focus should be trapped within the modal.
  • When the modal closes, focus should return to the element that triggered it.
  • The modal should use role="dialog" or role="alertdialog" for confirmation messages.
  • There must be a way to close the modal, such as a close button or the Escape key.

Best Practice

  • Providing a visible close button inside the modal improves usability.
  • Preventing background scrolling while the modal is open can reduce confusion.
  • Every modal should have a descriptive heading to clearly indicate its purpose.
