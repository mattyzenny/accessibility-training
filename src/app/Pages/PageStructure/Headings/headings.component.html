When to Use Headings

  • Use a heading when introducing a new section of content.
  • Each page should have exactly one <h1>.
  • Headings must be semantically marked up, not styled with CSS.
  • Headings should be sequentially ordered.

When Not to Use Headings

  • Do not use headings for visual styling. Use CSS instead.
  • Do not use <b> or <strong> to visually replace a heading.
  • Do not use headings as labels for form inputs. Use the <label> element.

Best Practices

  • Headings should be short, clear, and describe the section’s content.
  • Do not use more than one <h1> per page.
  • Ensure screen readers can navigate your page structure easily by following correct heading order.
  • Do not use aria-level. There is almost never a reason to use aria-level over HTML <h1>

Common Mistakes

  • ❌ Bad: Multiple MFE's sharing the same page may have non-sequential headings.
    ✅ Good: Coordinate heading levels across MFEs to maintain a logical structure.
  • ❌ Bad: Starting a page with the wrong heading level.
    ✅ Good: Ensure the main page <h1> is set by the shell app, and MFEs use <h2> or lower as needed.
  • ❌ Bad: Styling <div> to look like a heading.
    ✅ Good: Use a proper <h2> or <h3> tag.

            Another way to identify a missing heading is by looking at the CSS.
            We are visually embolding and changing the size of the font. We are
            applying a change to the text to draw the attention of the user, however we provide no such support for
            non-sighted users.
        

        

            To remain consistent, the <h1> should be applied on the first element,
            and the second element should be a <h2>.

            Lets look at a few of the mistakes on the heading structure.
        

        
    
            
  • Incorrect beginning header
    
                The page starts with an <h3> instead of <h1> or
                <h2>,
                skipping logical heading levels.
            
    • 
            
  • Missing heading for the section
                
    
                The User Name” section lacks a proper heading, making it harder for assistive technologies to identify
                its importance.
            
    • 
            
  • Incorrect use of semantic heading
                
    
                The “Profile Image” section uses a <div aria-level="2">, which is improper. <div> is not a
                valid heading element and should be replaced with an appropriatly descending <h2> or
                <h3>.
            
    • 
            
  • Heading order is inconsistent
                
    
                If following the page structure, the "Grades you teach" section should have a <h3>
                but is using <p>, breaking the expected hierarchy.
