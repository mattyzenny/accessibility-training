DO NOT USE NATIVE TOOLTIPS UNLESS THEY ARE CUSTOMIZED!

Let's clear the air on what a tooltip is. Historically and colloquially, a tooltip is a native browser popup that displays on hover. This is applied with the title attribute. We can already infer a few things wrong with this. A native HTML tooltip alone does not suffice in today's accessibility standards. A tooltip may be visible for individuals using a mouse, but anyone on a mobile device or using strictly keyboard navigation will never see these tooltips.

There is language that permits the title attribute despite its inherent inaccessibility, but for the purposes of this training, all content is expected to be accessible to all.

This means that we won't be discussing title and how it can pass if it is only providing supplemental content nor the innate inaccessibility of a user agent's and their inability to parse the title attribute on focus.

Now, using title is not wrong, and sometimes it can be useful — it just needs to be used sparingly. For instance, using a Native HTML tooltip on an icon button or icon link is a good practice. However, it is almost always preferred to provide a tooltip that displays both onHover and onFocus.

Warning Currently, cel-components does not support custom tooltips and only relies on the title attribute. We should migrate away from this practice.

Required

  • Icons must have a sufficient color contrast of 3:1. They must not break the contrast of surrounding colors. WCAG 1.4.11
  • Icons and images should not include text as part of the icon or image. WCAG 1.4.5
    1. Decorative
      • Icons must have an empty alt attribute. WCAG 1.1.1
    2. Non-Decorative
      • Icons must have a text alternative. WCAG 1.1.1
      • If interactive, they must have a tabindex WCAG 2.1.1
      • Use of color cannot be the only thing to convey meaning. WCAG 1.4.1
      • Icons must be consistently identified and used across the application. WCAG 3.2.3   WCAG 3.2.4
      • Content that appears on focus needs to be dismissible without moving the mouse. WCAG 1.4.13

Best Practice

  • Do not rely on the title attribute to provide alternative text. It should only be supplemental and non-required information.
  • Do not use the title attribute with an aria-label to avoid AT redundant announcement.
  • Provide visible text labels when possible. If you cannot, then at least provide a custom tooltips on ambiguous icons.
  • DO NOT rely on tooltips! Use them sparingly. If it is important enough for a popover, it should have its own space on the page.

Key Differences

  • Icon
    1. Icons do not inherently have a tabindex.
    2. If decorative, icons should have an empty alt attribute.
    3. Icons should never be used as a selectable item unless it is wrapped by an interactive element.
  • Icon-Button
    1. Icon-buttons inherently have a tabindex.
    2. Icon-buttons should perform some action.
    3. Cel-icon-buttons (currently) are not fully accessible, as they discriminate against keyboard users.
    4. Icon-buttons that use only the title attribute are not a good practice.
  • Icon-Button (custom)
    1. Icon-buttons should have a tabindex.
    2. Icon-buttons should perform some action.
    3. Icon-buttons, if ambiguous, should have tooltips on hover and focus.

Hover & Focus

Hover and tab to each element. When design wants an icon and not an icon button, that icon should be coupled with a text label.

If that icon is interactive, it must have a tabindex. Use <cel-icon-button> instead.

       Tooltip Icon with Popover
    

    

      Combining a tooltip with a popover is a great way to provide more information to the user.
      Let's discuss the practicality and complexity required for it.
    

    
    
      
  • 
        How will the popover look on a smaller viewport?
      
    • 
      
  • 
        How do we notify the user that the button opens a popover?
      
    • 
      
  • 
        How do we announce the content once the popover is displayed?
      
    • 
      
  • 
        Where do we move the focus on activation and on dismissal?
      
    • 
      
  • 
        Is there content inside the popover that needs styling or labeling?
      
    • 
    

    

      You may very well have an answer to all of these questions. It is certainly not wrong, and in some cases,
      it's necessary, but there are likely better solutions out there.
    

    

      Here are some guildeines.
    

    
    
      
  • 
        If its important, try to fit it on the page and not hidden inside a popover.
      
    • 
      
  • 
        If the content is already written somewhere else on the page, do not take page space with redundancy, use
        aria-describedby.
      
    • 
      
  • 
        Directions and labels should be concise and are a clearer way to provide information.
      
    • 
      
  • 
        If your content is succinct, but still contains too much text, only then is a popover appropriate.
      
    • 
    

    

      We must exercise extreme caution when using popovers. The more properties a component has, the greater the risk of
      introducing errors.

      Ambiguous Icons Buttons
    

    

      The icons below are not universally understood. It is crucial that designs consider the usability, practicality,
      and clarity of the icons we use.
      Not every user is a screen reader user, so not everyone will understand the function of the
      <cel-icon-button> just by looking at it.
    

    

      Tooltips are the hero in this scenario ➡️ where onHover() and
      onFocus() stand out.
    

  

  

    

      
      
      
    

  

  

    

      

        
      

      

        

          Icon Button

           data-color=
          "primary"
           data-name=
          "about-author"
           data-tooltip-label=
          "About Author"></cel-icon-button>
          


          Icon Button (custom)
          

          <cel-icon-button
           data-color=
          "primary"
           data-icon-button-aria-label=
          "About Author"
           data-name=
          "about-author"
           data-bs-tooltip=
          "true"
           data-bs-title=
          "About Author"></cel-icon-button>
