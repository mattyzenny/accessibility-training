Description lists (<dl>) are used to group related terms and descriptions. They provide a structured way to present information where multiple terms may be associated with a single description, or a single term may have multiple descriptions.

Screen readers interpret description lists by announcing the term (<dt>), followed by the description (<dd>). Users can navigate between pairs using standard list navigation commands.

Description lists should only be used for key-value relationships or glossary style information.

Required

  • The relationship between terms and descriptions must be meaningful and programmatically determinable. Use a <dl> only when the structure represents a **clear association** between terms and descriptions.
  • Each description term <dt> must have at least one corresponding <dd>. Similarly, each <dd> must be associated with a <dt>.
  • The name, role, and structure of the list must remain intact. Overriding native semantics with excessive styling or using role="presentation" on a <dl> can break accessibility.

Best Practices

  • A <dl> can be used when a table would be excessive for simple term-description pairs. It provides a structured and accessible way to associate terms (<dt>) with their descriptions (<dd>) without the complexity of a full table.
  • If the information requires tabular relationships, such as comparisons or multiple attributes per item, a <table> may be more appropriate.

In the example below, we have an HTML element that only represents key value pairs. When there is no need for a heading, and there is no need for table structure, we can code it as a description list.

                    <dl
                    class=
                    "override-container w-100">
                    

                      <div class="row border-bottom border-3 mb-3 pb-3">
                        <dt class="col-auto fw-bold">Description lists</dt>
                        <dd class="col text-end">A description list is perfect for defining terms.</dd>
                      </div>
                    

                    
                    <div class="row border-bottom border-3 mb-3 pb-3">
                    <dt class="col-auto fw-bold">Description list with 2 terms</dt>
                    <dt class="col-auto">
                    <cel-button
                    [buttonSize]
                    "buttonProps.size"
                    [label]=
                    "buttonProps.label"></cel-button>
                    </dt>
                    <dd class="col text-end">This is a definition for 2 terms.</dd>
                    </div>
                    

                    <div class="row border-bottom border-3 mb-3 pb-3">
                    <dt class="col-auto fw-bold">Another term</dt>
                    <dd class="col text-end">This definition is short, so no extra paragraphs or anything.</dd>
                    </div>
                    


                    <div class="row border-bottom border-3 pb-3">
                    <dt class="col-sm-1 fw-bold text-truncate"
                    data-bs-tooltip=
                    "true"
                    data-bs-title=
                    "Truncated term">
                    Truncated term</dt>
                    <dd class="col text-end">This can be useful when space is tight. Adds an ellipsis at the end.</dd>
                    <dd class="col text-end">Here is a second definition.</dd>
                    </div>
                    
                    


                    </dl>
