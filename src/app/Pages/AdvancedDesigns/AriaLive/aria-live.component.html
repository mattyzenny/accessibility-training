The first rule of ARIA: use it only when necessary. Native HTML elements should always come first—like using visible text inside a <button> instead of an aria-label. However, for complex web apps, ARIA can be essential for accessibility.

One powerful ARIA attribute is aria-live. Aria-live designates a live region on the page where screen readers automatically announce any updates that happen within that region. This is especially useful for dynamic content that changes on a page without shifting focus or a page refresh.

Required

  • Use aria-live on a container element that will receive dynamic updates, such as <div> or <span>.
  • Choose the appropriate aria-live token:
    • polite announces updates only when the user is idle.
    • assertive announces updates immediately, interrupting the user.
  • Ensure that content inside the live region is updated dynamically; static content won’t trigger announcements.

Best Practice

  • Keep live region updates short and relevant to avoid overwhelming users.
  • Avoid using aria-live="assertive" unless absolutely necessary, as it disrupts the user's flow.
  • Do not apply aria-live to large sections of the page; limit it to small, specific elements where updates occur.
  • Test with screen readers to ensure updates are announced as expected.
  • Aria-live does not work the same on all AT devices. Test with multiple screen readers to ensure compatibility.
  • Do not shift focus to the live region when updates occur; this can be disorienting for users.

Aria-live is a powerful tool with different tokens for different scenarios. The most frequently used is aria-live="polite", to learn about the other tokens and how to use them, visit the W3C ARIA Live Regions documentation.

Below is a demonstration of how aria-live works. In this scenario, we are dynamically removing content on the page. Once the content is removed, the screen reader will announce the change to the user.

  1. Turn on your screen reader.
  2. Add content into the required text field.
  3. Select the Clear Data button to trigger the live-region.

Try entering and modifying a message to see how assistive technologies handle live updates!
