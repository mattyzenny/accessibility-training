Checkbox groups, just like radio groups, require a label for each individual list item. A group label is also required. Each element must be interactive with both a keyboard and a mouse.

Radio groups and checkbox groups differ in navigation and selection. In a radio group, users tab into the group and use arrow keys to toggle 1 option at a time. In a checkbox group, users tab to each individual checkbox and can select multiple options.

Required

  • There must be a visible goup label.
  • Each selectable option must have a visible label.
  • The state of each element must be programmatically determinable.
  • Each element must be operable by keyboard.
  • The visible label must match the hidden label.
  • The state of each element must have a contrast ratio of 3:1.

Best Practice

  • Do not nest interactive elements inside a checkbox.
  • If the selection triggers dynamic content on the page...
    1. Ensure that the user is informed of the change.
    2. Have that dynamic content display outside of the checkbox group.
  • Labels should be visually adjacent.
  • If the field is required, ensure checkboxes or radio groups are visibly and programmatically marked as required.

Navigate to each element using tab. Use the spacebar to select or deselect each checkbox. If marked up correctly, when you enter a checkbox group, you will hear the checkbox label, the state, and the group name.
