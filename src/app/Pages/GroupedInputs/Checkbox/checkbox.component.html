Checkbox groups, just like radio groups, require a label for each individual list item. A group label is also required. Each element must be interactive with both a keyboard and a mouse.

The main difference between radio groups and checkbox groups is how a user navigates the elements. When tabbing into a radio group, you use arrow keys to choose different options. Generally 1 item in a radio group is active at a time. With checkbox groups, users can tab to each individual checkbox, and can select however many they want.

Required

  • Starting from the top level of the component, grouped inputs must have a visible, meaningful label <legend>. These are crucial for the understanding and purpose of the radio group. They can be seen as the instructions or the group label.
  • Similar to many other elements, each individual radio button must have a visible label. All users need to know what each button represents.
  • The name, role, state, & value must be programmatically determinable. (role is often baked into the native HTML <button> component | name is the visible text | value, state are required for more complex functionality)

This first radio group is using the bare mininmum to be accessible. There is a group label, and each radio button has a visible label. Only 2 props are required to pass accessibility when using cel-components.

When you select an input, you will hear the state of "selected" be announced. In the second input of the second radio group, the state of "disabled" is announced.
